Materi SQL/MySQL

1. Pengertian SQL dan MySQL

SQL (Structured Query Language) adalah bahasa yang digunakan untuk berinteraksi dengan database, seperti untuk menambah, menghapus, dan mengambil data. SQL adalah standar untuk manajemen database relasional.

MySQL adalah sistem manajemen database (DBMS) berbasis SQL yang open-source, digunakan secara luas dalam aplikasi web. MySQL dikelola oleh Oracle Corporation.

2. Dasar-Dasar SQL

a. Perintah Dasar SQL

  • SELECT: Digunakan untuk mengambil data dari tabel. 
    SELECT * FROM users;
  • INSERT: Digunakan untuk menambah data ke dalam tabel. 
    INSERT INTO users (nama, umur) VALUES ('John', 25);
  • UPDATE: Digunakan untuk memperbarui data dalam tabel. 
    UPDATE users SET umur = 26 WHERE nama = 'John';
  • DELETE: Digunakan untuk menghapus data dari tabel. 
    DELETE FROM users WHERE nama = 'John';

3. Tabel dan Relasi

Di MySQL, data disimpan dalam tabel yang terdiri dari baris dan kolom. Relasi antar tabel dapat dibentuk dengan menggunakan kunci asing (foreign key).

  • Membuat Tabel: 
    CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nama VARCHAR(100),
  umur INT
);
  • Relasi Antar Tabel: Membuat relasi antara tabel `users` dan `orders`. 
    CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT,
  total DECIMAL(10, 2),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

4. Manipulasi Data

MySQL memungkinkan kita untuk memanipulasi data dalam berbagai cara, seperti menggunakan fungsi agregasi atau melakukan operasi pencarian.

  • Fungsi Agregasi: Fungsi seperti COUNT, SUM, AVG, MIN, dan MAX. 
    SELECT COUNT(*) FROM orders;
SELECT AVG(total) FROM orders;
  • WHERE Clause: Untuk memfilter data berdasarkan kondisi. 
    SELECT * FROM users WHERE umur > 18;
  • ORDER BY: Untuk mengurutkan data berdasarkan kolom tertentu. 
    SELECT * FROM users ORDER BY umur DESC;

5. Join dan Relasi Antar Tabel

Untuk mengambil data dari beberapa tabel yang berelasi, kita dapat menggunakan perintah JOIN.

  • INNER JOIN: Mengembalikan data yang sesuai dari kedua tabel. 
    SELECT users.nama, orders.total
FROM users
INNER JOIN orders ON users.id = orders.user_id;
  • LEFT JOIN: Mengembalikan semua data dari tabel kiri meskipun tidak ada pasangan di tabel kanan. 
    SELECT users.nama, orders.total
FROM users
LEFT JOIN orders ON users.id = orders.user_id;

6. Keamanan Database

Dalam mengelola database, keamanan sangat penting untuk mencegah akses yang tidak sah dan melindungi data sensitif.

  • Pengguna dan Hak Akses: MySQL memungkinkan kita untuk mengelola pengguna dan hak akses. 
    CREATE USER 'admin'@'localhost' IDENTIFIED BY 'password123';
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost';
  • Backup dan Restore: Backup data sangat penting untuk mencegah kehilangan data. 
    -- Backup Database
mysqldump -u root -p nama_database > backup.sql;

-- Restore Database
mysql -u root -p nama_database < backup.sql;

7. Index dan Optimasi

Index digunakan untuk mempercepat proses pencarian data di tabel besar. Namun, pembuatan terlalu banyak index dapat memperlambat proses penulisan (INSERT, UPDATE, DELETE).

  • Membuat Index: 
    CREATE INDEX idx_nama ON users(nama);
  • Optimasi Query: Hindari penggunaan SELECT * dan gunakan query yang hanya memanggil kolom yang diperlukan. 
    SELECT nama FROM users WHERE umur > 20;

8. Subquery dan Nested Query

Subquery adalah query di dalam query lain yang digunakan untuk melakukan pencarian lebih kompleks.

  • Contoh Subquery: Menampilkan user dengan total order lebih besar dari rata-rata. 
    SELECT nama FROM users WHERE id IN 
  (SELECT user_id FROM orders WHERE total > (SELECT AVG(total) FROM orders));

9. Transaction dan ACID

Transaction adalah sekelompok operasi SQL yang dieksekusi sebagai satu kesatuan. Konsep ACID (Atomicity, Consistency, Isolation, Durability) menjaga keandalan transaksi.

  • Memulai Transaksi: 
    START TRANSACTION;
  • Commit dan Rollback: Commit untuk menyimpan perubahan, Rollback untuk membatalkan. 
    COMMIT;  -- Untuk menyimpan
  ROLLBACK;  -- Untuk membatalkan

10. Fungsi dan Stored Procedure

Fungsi dan stored procedure digunakan untuk mengeksekusi blok kode SQL yang dapat dipanggil berulang kali. Stored procedure dapat menerima parameter dan mengembalikan nilai.

  • Membuat Fungsi: 
    CREATE FUNCTION hitung_umur(tanggal_lahir DATE) RETURNS INT
  BEGIN
    DECLARE umur INT;
    SET umur = YEAR(CURDATE()) - YEAR(tanggal_lahir);
    RETURN umur;
  END;
  • Stored Procedure: 
    CREATE PROCEDURE tambah_user(IN nama_user VARCHAR(100), IN umur_user INT)
  BEGIN
    INSERT INTO users (nama, umur) VALUES (nama_user, umur_user);
  END;

Kesimpulan

MySQL merupakan salah satu DBMS yang paling populer dan handal dalam pengelolaan database. Dengan memahami dasar-dasar SQL, manipulasi data, relasi antar tabel, serta konsep lanjutan seperti transaction, index, dan keamanan, kita dapat mengelola database dengan lebih efisien dan aman. Melalui penggunaan query yang optimal, kita juga dapat meningkatkan performa aplikasi yang bergantung pada MySQL.
